---
title: useIsUsernameAvailable
description: Check whether a username is free to claim.
---

Modeled as a mutation (not a query) because callers typically trigger the check on user action — a debounced input or form submit — rather than on mount.

`data` is typed as `{ available: boolean; message: string | null }`.

## Usage

```tsx
import { useAuth } from "@better-auth-ui/react"
import {
  type UsernameAuthClient,
  useIsUsernameAvailable
} from "@better-auth-ui/react/plugins/username"

const { authClient } = useAuth<UsernameAuthClient>()
const { mutateAsync: checkUsername, data } = useIsUsernameAvailable(authClient)

const result = await checkUsername({ username: "alice" })
// result.available -> boolean
```

`useIsUsernameAvailable` requires the Better Auth `username` plugin. Type `useAuth` with `UsernameAuthClient` so TypeScript picks up the plugin-extended method signature.

## Options factory

```tsx
import { isUsernameAvailableOptions } from "@better-auth-ui/react/plugins/username"
import { useMutation } from "@tanstack/react-query"

const { mutate } = useMutation(isUsernameAvailableOptions(authClient))
```
