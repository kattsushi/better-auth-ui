`@better-auth-ui/solid` is the SolidJS track for Better Auth UI. It keeps the same core keys, auth configuration defaults, and plugin contracts as the React package where Solid supports equivalent behavior.

## Install

```package-install
@better-auth-ui/solid @tanstack/solid-query better-auth solid-js
```

The package exports only the Solid-supported surface:

- `@better-auth-ui/solid` — provider, hooks/composables, query factories, mutation factories.
- `@better-auth-ui/solid/server` — server-side session helpers.
- `@better-auth-ui/solid/plugins` — plugin config helpers aligned with shared core contracts.

React-only email templates and toast UI are intentionally not exported from the Solid package.

## Quick path

1. Install `@better-auth-ui/solid` with `@tanstack/solid-query`.
2. Use the Solid Start reference app at `examples/start-solid-zaidan-example` for route and provider wiring.
3. Install Zaidan registry components from the `/r/solid` namespace when component payloads are needed.
4. Check [Gaps](/docs/solid/gaps) before assuming React package parity.
