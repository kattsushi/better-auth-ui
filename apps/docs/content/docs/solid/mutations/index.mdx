---
title: Mutations
description: Solid mutation factories for Better Auth write endpoints.
---

Solid mutations expose `@tanstack/solid-query` mutation options for Better Auth write endpoints. They preserve shared mutation keys and pass `throw: true` so failures surface through standard Solid Query error handling.

Unlike React, Solid package mutations are currently **options factories**, not purpose-built `use*` mutation hooks. Use the factory with `createMutation`, or use the generic `useAuthMutation` escape hatch for endpoints without a named factory.

```tsx
import { signInEmailOptions } from "@better-auth-ui/solid"
import { createMutation } from "@tanstack/solid-query"

const signIn = createMutation(() => signInEmailOptions(authClient))
```

## Error handling

Every named factory wires `throw: true` into `fetchOptions`, so the mutation rejects with a `BetterFetchError` on failure instead of resolving with `{ error }`.

## Cache side effects

Solid factories intentionally do not attach React-style hook side effects. Invalidate or refetch explicitly with the shared keys after a successful mutation.

```ts
import { authQueryKeys } from "@better-auth-ui/core"

await queryClient.invalidateQueries({ queryKey: authQueryKeys.session })
```

## Escape hatch

For write-style endpoints without a purpose-built factory, use `useAuthMutation` in components or `authMutationOptions` where Solid Query takes mutation options.

## UI boundary

The Solid package does not render toast UI. Attach success/error handling in your app, or use installed Zaidan components that already integrate with the app's toast surface.
