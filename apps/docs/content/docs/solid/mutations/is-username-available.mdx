title: isUsernameAvailableOptions
description: Check username availability.
---

Requires the Better Auth username plugin. When reading the client from auth context, call `useAuth<UsernameAuthClient>()` so TypeScript sees the plugin methods.

This check is modeled as a mutation because callers usually trigger it from user input, not from route-level data loading. Returned data is typed as `{ available: boolean; message: string | null }`.

## Options factory

```tsx
import { isUsernameAvailableOptions } from "@better-auth-ui/solid/plugins/username"
import { createMutation } from "@tanstack/solid-query"

const checkUsername = createMutation(() => isUsernameAvailableOptions(authClient))

checkUsername.mutate({ username: "alice" })
```
