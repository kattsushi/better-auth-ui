Solid plugin docs describe package/runtime contracts, not installable component catalogs.

## Quick path

1. Enable Better Auth plugins in your app server config.
2. Enable matching Better Auth client plugins in your Solid client config.
3. Use Solid query and mutation factories from `@better-auth-ui/solid` where the plugin exposes runtime endpoints.
4. Use [Zaidan plugins](/docs/zaidan/plugins) only when you need copied UI files.

## Package export boundary

`@better-auth-ui/solid/plugins` is a type-only export:

```ts
export type { AuthPlugin } from "@better-auth-ui/core"
```

It does not ship runtime plugin helpers. Runtime plugin composition still comes from Better Auth, `@better-auth-ui/core/plugins`, or app-local Solid adapters when a plugin needs Solid UI registration.

## Runtime capabilities

| Capability | Solid package API |
| --- | --- |
| Username | `signInUsernameOptions` and `isUsernameAvailableOptions`. |
| Magic link | `signInMagicLinkOptions`. Replace development email handlers with a production email provider in app code. |
| Passkey | `listPasskeysOptions`, `addPasskeyOptions`, `deletePasskeyOptions`, and `signInPasskeyOptions`. Configure WebAuthn origin/trusted origins in Better Auth. |
| Multi-session | `listDeviceSessionsOptions`, `setActiveSessionOptions`, and `revokeMultiSessionOptions`. |
| API key | `listApiKeysOptions`, `createApiKeyOptions`, and `deleteApiKeyOptions`. |
| Delete user | `deleteUserOptions`; the Better Auth server config still owns the actual delete-user capability. |
| Social provider | `signInSocialOptions`, `linkSocialOptions`, and `unlinkAccountOptions`; provider secrets stay in app/server environment config. |

## Caveats

- Captcha is out of scope for the Solid package widget contract in this slice. Server-side Better Auth captcha config can still protect endpoints, but a Solid UI token/reset lifecycle needs a separate design task.
- Theme switching is app/UI behavior. The Solid package does not export a theme plugin.
- Installable plugin screens are documented under [Zaidan plugins](/docs/zaidan/plugins), because those files are copied into your app and become app-owned.
