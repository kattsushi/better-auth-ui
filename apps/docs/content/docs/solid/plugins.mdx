---
title: Plugins
description: Plugin registration notes for the Solid package.
---

Solid plugin support follows the shared `@better-auth-ui/core` plugin contracts. Use Better Auth plugins in your app's server/client auth config, then wire Solid query and mutation factories from `@better-auth-ui/solid` in your UI.

`@better-auth-ui/solid/plugins` is currently a type-only export:

```ts
export type { AuthPlugin } from "@better-auth-ui/core"
```

It does not ship runtime plugin helpers yet. If a plugin needs UI, install or create the Solid/Zaidan component for that feature in your app or registry source.

## Supported parity

| Area | Package status | Example/registry status | Notes |
|------|----------------|-------------------------|-------|
| Captcha | Shared config types only | Not shown | Use the same core captcha options as React-compatible flows; no Solid/Zaidan captcha UI is delivered yet. |
| Passkey | Query and mutation factories | Settings list shell shown; dialogs still pending | Factories are available for list/add/delete/sign-in flows. |
| Multi-session | Query and mutation factories | Account/session rows shown; switch/revoke actions still pending | Query and mutation keys align with the core namespace. |
| API key | Query and mutation factories | Settings list shell shown; create/delete dialogs still pending | API-key query/mutation factories use shared plugin key contracts. |
| Magic link | Mutation factory | Not shown in the current Solid registry | Mutation factories preserve Better Auth fetch error behavior. |
| Username | Mutation factories | Sign-in route supports username/email selection | Username sign-in and availability flows use shared mutation keys. |

## Non-goals

`react-email`, `sonner`, and HeroUI React component integrations are not Solid plugin exports. Track these in [Gaps](/docs/solid/gaps) instead of assuming silent parity.
