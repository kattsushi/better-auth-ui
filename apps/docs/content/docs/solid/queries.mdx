title: Queries
description: Solid Query factories and auth read helpers.
---

Solid queries are package runtime APIs. They return `@tanstack/solid-query` options and hooks for Better Auth read endpoints, with shared `@better-auth-ui/core` cache keys.

## Quick path

1. Pick the exported factory that matches the Better Auth client method.
2. Pass your app's typed `authClient`.
3. Use the `*Options` function in loaders/prefetch code or the `use*` hook in Solid components.

```ts
import { sessionOptions, useSession } from "@better-auth-ui/solid"

const sessionQuery = sessionOptions(authClient)
const session = useSession(authClient, { enabled: !import.meta.env.SSR })
```

## Session helpers

| Helper | Use when |
| --- | --- |
| `sessionOptions` | You need raw Solid Query options. |
| `ensureSession` | A loader must synchronously ensure session data in the query cache. |
| `prefetchSession` | A route can warm the cache without reading the value immediately. |
| `fetchSession` | You need one explicit fetch through the query client. |
| `useSession` | A Solid component needs reactive session state. |

## User-scoped query families

These factories use the active user/session boundary and shared cache keys instead of inline tuples:

| Area | Exported helpers |
| --- | --- |
| Settings | `accountInfoOptions`, `listAccountsOptions`, `listSessionsOptions`, plus `ensure*`, `prefetch*`, `fetch*`, and `use*` variants. |
| Passkey | `listPasskeysOptions`, `ensureListPasskeys`, `prefetchListPasskeys`, `fetchListPasskeys`, `useListPasskeys`. |
| Multi-session | `listDeviceSessionsOptions`, `ensureListDeviceSessions`, `prefetchListDeviceSessions`, `fetchListDeviceSessions`, `useListDeviceSessions`. |
| API key | `listApiKeysOptions`, `ensureListApiKeys`, `prefetchListApiKeys`, `fetchListApiKeys`, `useListApiKeys`. |

## Generic primitives

`authQueryOptions` adapts a Better Auth read method to Solid Query options. `createUserScopedOptions`, `ensureUserScopedQuery`, `prefetchUserScopedQuery`, `fetchUserScopedQuery`, and `createUserScopedQuery` are lower-level primitives used by package query families.

Reach for the named helpers first. Use the generic primitives only when you are adding a new package query family.

## Cache keys

Use shared core keys instead of inlining tuples.

```ts
import { authQueryKeys } from "@better-auth-ui/core"

queryClient.invalidateQueries({ queryKey: authQueryKeys.session })
queryClient.invalidateQueries({ queryKey: authQueryKeys.user(userId) })
```
