---
title: <Auth />
description: A composed Solid auth route component that renders the matching auth view for the current route path.
---

## Usage

Use `<Auth />` from your Solid auth route and pass the route segment from TanStack Router.

```tsx
import { Auth } from "@/components/auth/auth"

export function AuthRoute(props: { params: { path: string } }) {
  const params = props.params

  return <Auth path={params.path} />
}
```

The component resolves sign-in, sign-up, sign-out, forgot-password, and reset-password views from the installed `AuthProvider` context. Unsupported paths render nothing so your route can keep its own fallback or redirect policy.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/auth.json
```

## Props

| Prop | Type | Description |
| --- | --- | --- |
| `path` | `string` | Optional auth route segment, usually `params.path` from `/auth/$path`. |

## Installed files

Installs the auth router surface that composes the auth flow views and route-path helpers.

- `src/components/auth/auth.tsx`

After install, keep allowed auth paths aligned with the plugin views you enable.
