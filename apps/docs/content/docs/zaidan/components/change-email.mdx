## Usage

Render `ChangeEmail` in account settings when users can update their email.

```tsx
import { ChangeEmail } from "@/components/auth/settings/account/change-email"

export function EmailCard() {
  return <ChangeEmail />
}
```

The card reads the current session email, submits through `changeEmailOptions`, and uses the configured settings account path as the callback URL.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/change-email.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. Configure auth client, localization, base URL, and settings paths through `AuthProvider`.

## Plugin gates

- Requires an authenticated session.
- Uses `changeEmailOptions` from `@better-auth-ui/solid`.
- If your Better Auth setup requires email verification, make sure the callback URL and email provider are configured before exposing the card.
