## Usage

Render `ChangePasswordSettings` when email/password auth is enabled.

```tsx
import { ChangePasswordSettings } from "@/components/auth/settings/security/change-password"

export function PasswordCard() {
  return <ChangePasswordSettings confirmPassword />
}
```

If the signed-in user has no `credential` account, the card sends a password reset link instead of showing the current-password form.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/change-password.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. `confirmPassword` remains a Zaidan/Solid-adapted option because this copied card reads the Better Auth password policy from local context.

## Plugin gates

- `emailAndPassword` must be enabled before `SecuritySettings` renders this card.
- A linked `credential` account determines whether the change-password form or set-password flow appears.
- Uses `requestPasswordResetOptions` for set-password emails and `changePasswordOptions` for credential changes.
