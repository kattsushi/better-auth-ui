Zaidan component docs map to real `/r/solid/*.json` payloads generated from the Solid Start example.

## Install model

1. Install `auth-provider` first.
2. Add the auth, user, or settings payload you need.
3. Review copied files and customize them in your app.

```bash
zaidan add https://better-auth-ui.com/r/solid/auth-provider.json
zaidan add https://better-auth-ui.com/r/solid/sign-in.json
zaidan add https://better-auth-ui.com/r/solid/settings.json
```

## Groups

- **Provider**: `auth-provider` wires the copied app provider shell.
- **Shared**: `additional-field` is reused by sign-up and profile forms.
- **Auth**: `auth`, `sign-in`, `sign-up`, `forgot-password`, `reset-password`, `sign-out`.
- **User**: `user-button`, `user-avatar`, `user-view`.
- **Settings**: `settings`, `account-settings`, `user-profile`, `change-email`, `security-settings`, `active-sessions`, `linked-accounts`, `change-password`.

Email templates are intentionally out of scope because Solid/Zaidan does not ship React email payloads.
