## Usage

Render `LinkedAccountsSettings` when your app exposes social providers.

```tsx
import { LinkedAccountsSettings } from "@/components/auth/settings/security/linked-accounts"

export function SocialAccountsCard() {
  return <LinkedAccountsSettings />
}
```

The card combines existing linked accounts with configured providers so users can connect or disconnect accounts such as GitHub.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/linked-accounts.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. Providers still come from `AuthProvider` via Better Auth UI configuration.

## Plugin gates

- `socialProviders` controls whether `SecuritySettings` renders this card.
- Uses `listAccountsOptions` to load linked accounts outside SSR.
- Configure social providers such as GitHub in Better Auth before exposing link/unlink actions.
