## Usage

Use `SecuritySettings` inside your settings shell when the active tab is security.

```tsx
import { SecuritySettings } from "@/components/auth/settings/security/security-settings"

export function SecurityTab() {
  return <SecuritySettings />
}
```

The component renders only the cards supported by your Better Auth config and installed payloads.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/security-settings.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. Plugin-gated cards continue to be controlled by your Better Auth configuration.

## Plugin gates

- `emailAndPassword` shows password management.
- `socialProviders` shows linked account management.
- `apiKey` shows API key management when the API key plugin is installed.
- `passkey` shows passkey management when the passkey plugin is installed.
- `deleteUser` shows the danger-zone card when the delete-user plugin is installed.

## Installed files

- `src/components/auth/settings/security/security-settings.tsx`
- `src/components/auth/settings/shared/helpers.ts`
- `src/components/auth/settings/shared/types.ts`

After install, add plugin payloads such as passkey and api-key only when the matching Better Auth plugins are configured.
