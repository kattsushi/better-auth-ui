## Usage

Mount `Settings` from a protected TanStack Start route and pass the active settings path.

```tsx
import { Settings } from "@/components/auth/settings/settings"

function SettingsPage() {
  const path = () => Route.useParams()().path

  return (
    <div class="mx-auto w-full max-w-3xl p-4 md:p-6">
      <Settings path={path()} />
    </div>
  )
}
```

The route should require a session before rendering this surface. The shell maps the configured settings paths to the account and security tabs and redirects unknown paths at the route layer.

## Parity classification

Zaidan follows Solid's local `class` prop convention. It adapts the shadcn `className` styling hook to `class`; `path`, `view`, and `hideNav` keep their shadcn semantics in Solid form.

## Plugin gates

`Settings` itself only needs an authenticated session. The account and security panels decide which plugin-backed cards appear, so install the child payloads that match your app configuration.

## Installed files

- `src/components/auth/settings/settings.tsx`
- Local tabs, card, separator, and menu primitives.

After install, keep the route guard and the account/security child payloads aligned with your Better Auth config.
