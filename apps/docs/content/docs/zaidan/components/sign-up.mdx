---
title: <SignUp />
description: A Solid sign-up form with name, email, password, username, provider buttons, and additional-field support.
---

## Usage

Render `<SignUp />` inside an auth route wrapped by the installed `<AuthProvider />`.

```tsx
import { SignUp } from "@/components/auth/sign-up"

export function SignUpRoute() {
  return <SignUp />
}
```

The component reads email/password policy, username plugin state, additional fields, provider buttons, and navigation paths from the Solid auth context.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/sign-up.json
```

## Props

`<SignUp />` does not expose public props. Configure name fields, confirmation password behavior, verification flow, username support, and provider buttons through `<AuthProvider />` and your Better Auth setup.

## Installed files

Installs the email sign-up form with confirmation password feedback, provider buttons, username support, and additional-field rendering.

- `src/components/auth/sign-up.tsx`
- `src/components/auth/provider-button.tsx`
- `src/components/auth/provider-buttons.tsx`
- Local form UI primitives.

After install, review password policy, verification messaging, and additional-field requirements.
