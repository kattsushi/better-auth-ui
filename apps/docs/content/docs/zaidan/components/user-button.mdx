## Usage

Install `user-button` when you need the signed-in account menu in a Solid route or header. It reads the active session from `AuthProvider`, shows sign-in/sign-up actions when unauthenticated, and renders the settings, theme, and sign-out entries when a user is signed in.

```tsx
import { UserButton } from "@/components/auth/user/user-button"

export function HeaderActions() {
  return <UserButton size="icon" align="end" />
}
```

Use the default size when the trigger should show the avatar, user label, and secondary label together.

```tsx
import { UserButton } from "@/components/auth/user/user-button"

export function AccountMenu() {
  return <UserButton />
}
```

The menu uses the copied theme item from the Zaidan payload and the route paths from your `AuthProvider` config. It displays the current Better Auth session only; if your app enables multi-session, keep account switching in the multi-session/settings payload and use this button as the current-session entry point.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/user-button.json
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `class` | `string` | `undefined` | Extra class applied to the trigger button. |
| `align` | `"center" \| "end" \| "start"` | `"center"` | Aligns the dropdown content relative to the trigger. |
| `sideOffset` | `number` | `4` | Gap between the trigger and dropdown content. |
| `size` | `"default" \| "icon"` | `"default"` | Renders either the full account row or the avatar-only trigger. |
| `variant` | `"default" \| "destructive" \| "ghost" \| "link" \| "outline" \| "secondary"` | `"ghost"` | Button variant forwarded to the local Zaidan button primitive. |

## Installed files

After install, point the auth and settings links at your TanStack Router paths and decide whether the header should render the icon or full account row.
