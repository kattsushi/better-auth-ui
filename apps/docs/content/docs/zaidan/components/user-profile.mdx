## Usage

Render `UserProfile` inside account settings once your app is wrapped in `AuthProvider`.

```tsx
import { UserProfile } from "@/components/auth/settings/account/user-profile"

export function ProfileCard() {
  return <UserProfile />
}
```

The form reads the session user, calls `updateUserOptions`, and keeps avatar UI in the copied `ChangeAvatar` file so you can wire uploads to your storage layer.

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/user-profile.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. Profile fields, username support, and avatar changes remain driven by installed plugins and provider configuration.

## Plugin gates

- `session user` data supplies the current name, email, and image.
- `username` support is used when the signed-in user has a username field.
- `avatar` upload behavior is app-owned after install; connect `ChangeAvatar` to your upload/storage flow.

## Installed files

- `src/components/auth/settings/account/user-profile.tsx`
- `src/components/auth/settings/account/change-avatar.tsx`
- `src/components/auth/additional-field.tsx`

After install, adapt additional fields and avatar persistence to your account model.
