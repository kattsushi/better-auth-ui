title: <UserView />
description: Renders a compact Solid user identity row with avatar, primary label, and optional secondary label.
---

## Usage

Use `UserView` inside account menus, settings headers, or custom session surfaces where you already have the session user data. It composes the copied `UserAvatar` component and leaves fetching to the parent surface.

```tsx
import { UserView } from "@/components/auth/user/user-view"

export function AccountSummary() {
  return <UserView label="Ada Lovelace" secondaryLabel="ada@example.com" />
}
```

When you render it from session state, pass the user's image and a short fallback from the active Better Auth session.

```tsx
import { UserView } from "@/components/auth/user/user-view"

export function SessionUserView(props: {
  user: { email?: string | null; image?: string | null; name?: string | null }
}) {
  const label = () => props.user.name || props.user.email || "Account"

  return (
    <UserView
      image={props.user.image}
      initials={label().slice(0, 2).toUpperCase()}
      label={label()}
      secondaryLabel={props.user.email}
    />
  )
}
```

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/user-view.json
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `image` | `string \| null` | `undefined` | Session user image URL forwarded to `UserAvatar`. |
| `initials` | `string` | `undefined` | Fallback text shown when no image can render. |
| `label` | `string` | Required | Primary visible user label. Use name, username, or email. |
| `secondaryLabel` | `string \| null` | `undefined` | Optional secondary line, commonly the user's email. |

## Installed files

- `src/components/auth/user/user-view.tsx`
- `src/components/auth/user/user-avatar.tsx`
- `src/components/ui/avatar.tsx`

After install, tune the label and secondary label mapping to match your account model.
