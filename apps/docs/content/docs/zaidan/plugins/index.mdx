Zaidan plugin docs cover copied UI payloads. Package-level plugin contracts stay in [Solid plugins](/docs/solid/plugins).

## Quick path

1. Enable the Better Auth server/client plugin in your app.
2. Install the matching Zaidan payload.
3. Customize the copied files in your app.
4. Return to [Solid plugins](/docs/solid/plugins) when you need package query/mutation APIs.

## Install model

Plugin payloads copy UI and local plugin helpers into your app. They do not enable Better Auth plugins for you.

```bash
zaidan add https://better-auth-ui.com/r/solid/passkey.json
zaidan add https://better-auth-ui.com/r/solid/api-key.json
```

Social/GitHub is provider-button support, not a separate payload. Enable your GitHub provider in Better Auth and install auth payloads such as `sign-in` or `sign-up`.

Captcha is pending/out of scope for this Zaidan phase. Do not treat React captcha docs as Solid/Zaidan support.

After install, copied files are yours. Keep runtime plugin questions in [Solid plugins](/docs/solid/plugins), and keep registry install questions in this Zaidan section.
